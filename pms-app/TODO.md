## 3. 기존 페이지 기능 개선

### 3-1. 자기 평가 (`/performance`) — 현황 확인 필요

현재 `/performance`와 `/evaluation` (MemberEvalView)에 자기평가 기능이 **두 곳에** 존재함.

- `/performance`: `YearEndEval` 컬렉션, 과제업무 세부요약 작성
- `/evaluation` MemberEvalView: `SelfEvaluation` 컬렉션, 목표별 잘된점/아쉬운점 작성

**조치**: 두 흐름 중 어느 것을 메인으로 쓸지 결정 후 통합 또는 역할 분리 필요.
기획서 기준: "과제업무 의견 + 일반업무 의견" 두 가지 입력 → `/performance`의 구조가 더 가까움.

### 3-2. 팀원평가 (`/evaluation` TEAM_LEAD 뷰) 개선

| 기능 | 현재 상태 | 조치 |
|------|-----------|------|
| 팀원 등급 선택 + 의견 작성 | ✅ | - |
| **팀원 육성면담서 확인** | ❌ | 팀원의 MentoringForm 데이터 조회 뷰 추가 |
| **팀원 육성면담서 팀장 의견 작성** | ❌ | MentoringForm 또는 별도 필드에 팀장 의견 저장 |

### 3-3. 평가등급 확정 (`/evaluation` EXECUTIVE 뷰) 개선

| 기능 | 현재 상태 | 조치 |

| **팀장 자기평가 확인** | ❌ | 확인 UI 추가 |

| **팀장 육성면담서 확인 및 임원 의견** | ❌ | 추가 |
| **팀원 육성면담서 확인 및 임원 의견** | ❌ | 추가 |
