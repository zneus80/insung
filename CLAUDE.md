## 1. 개요

팀원·팀장·임원·최고관리자·HR관리자 5개 역할로 구성된 사내 평가 플랫폼.
조직 단위 등급 설정 → 인원 배정 → 자기 평가 → 팀원 평가 → 평가등급 확정까지 전 과정을 시스템에서 처리한다.
평가 기간은 HR관리자가 설정하며, 목표관리 시스템과 연동하고 연도별 평가 이력을 관리한다.

모든 사이드바 항목은 이미 개발완료 상태이나, 현재 직급 구분 없이 전체 노출되어 있어
**역할별 필요 항목만 노출되도록 조정이 필요하다.**

## 2. 역할 정의

| 역할 | 설명 |
|------|------|
| 최고관리자 | 조직별 평가 등급 결정 (최종 권한) |
| HR관리자 | 별도 직급이 아닌 **권한 추가 개념**. 팀원/팀장 등 본인 직급에 HR관리자 권한이 추가됨. 평가 기간 설정, 결과 공개, 인원 배정 담당 |
| 임원 (최상위) | 소관 조직 평가등급 최종 확정 권한. 복수 조직 담당 가능 |
| 임원 (차순위) | 동일 조직 내 직급 차순위 임원. 팀원 평가·자기 평가 권한만 보유 (팀장과 동일) |
| 팀장 | 팀원 평가 제출, 자기 평가 작성, 평가결과 확인 |
| 팀원 | 자기 평가 작성, 평가결과 확인 |

### 임원 권한 케이스 정의

**케이스 A — 임원 1명이 복수 조직 담당**
- 해당 임원이 담당 조직 전체의 평가등급 확정 권한 보유
- 조직별로 독립적으로 확정 처리

**케이스 B — 1개 조직에 임원 복수 배치**
- 최상위 직급 임원 → **임원 (최상위)** 권한: 평가등급 확정 가능
- 차순위 직급 임원 → **임원 (차순위)** 권한: 팀원 평가 + 자기 평가만 가능 (확정 불가)

> ⚠️ **개발 주의**: 임원 역할 구분은 계정 단위가 아닌 **조직 배정 단위**로 관리.
> 동일 인물이 조직 A에서는 최상위 임원, 조직 B에서는 차순위 임원이 될 수 있음.

## 3. 평가 프로세스 흐름

```
① 조직평가관리  →  ② 조직평가인원관리  →  ③ 자기 평가         →  ④ 팀원 평가        →  ⑤ 평가등급 확정
  (최고관리자)        (HR관리자)            (팀장/팀원/             (팀장/                  (임원 최상위,
                                            차순위임원,             차순위임원,              소관 조직)
                                            과제/일반업무,          등급선택+의견작성,
                                            목표관리 연동)          임원에 자동 전달)
```

### 자동 전달 규칙

- 팀원 자기 평가 제출 → 팀장(또는 차순위 임원)의 **팀원 평가**로 자동 전달
- 팀장·차순위임원 자기 평가 제출 → 최상위 임원의 **평가등급 확정**으로 자동 전달
- 팀장·차순위임원 팀원 평가 제출 → 최상위 임원의 **평가등급 확정**으로 자동 전달

## 4. 역할별 사이드바 노출 항목

> 모든 항목은 개발완료 상태. 현재 직급 구분 없이 전체 노출 → 역할별 필요 항목만 노출되도록 제어 필요.
> HR관리자는 별도 직급이 아니므로 본인 직급 사이드바 항목 + HR관리자 추가 항목이 합쳐져 노출됨.

### 팀원
- 대시보드
- 목표관리
- 자기평가
- 육성면담서
- 평가결과확인
- 1on1

### 팀장
- 대시보드
- 목표관리 (세부: 내 목표 진행현황 + 팀원목표)
- 승인대기함
- 자기평가
- 팀원평가
- 육성면담서
- 평가결과확인
- 1on1

### 임원
- 대시보드
- 진행현황 (세부: 팀장 업무 진행사항 + 팀원 업무 진행사항)
- 승인대기함
- 평가등급확정
- 1on1

### 최고관리자
- 대시보드
- 진행현황 (세부: 조직 전체 업무진행현황)
- 조직평가관리

### HR관리자 (권한 추가 — 본인 직급 사이드바에 아래 항목이 추가됨)
- 조직평가인원관리
- 평가기간 관리
- 평가이력 관리
- 사용자 관리
- 조직 관리
- 연간목표 관리
- 마일리지 관리
- 시스템 설정

> 예시: 팀장이 HR관리자로 지정된 경우 → 팀장 사이드바 8개 항목 + 위 8개 항목 = 총 16개 항목 노출

## 6. 평가 기간 및 결과 공개

### 평가 기간 설정 — HR관리자
- 평가 시작일·종료일을 HR관리자가 직접 설정
- 기간 내에만 각 단계 작성 가능

### 평가결과 공개 — HR관리자 일괄 공개
- 평가등급 확정 완료 후, HR관리자가 공개 시점을 별도 설정하여 일괄 오픈
- 확정 즉시 자동 공개되지 않음
- 공개 전까지 팀장·팀원·차순위임원은 평가결과 확인 메뉴 접근 불가 (또는 비공개 상태 표시)

## 9. 개발 주요 고려사항

1. **전체 기능 확인필요** — 개발완료 항목과 미개발 항목 혼재. 코드 기준으로 각 기능 구현 여부 먼저 확인 후 작업
2. **역할별 사이드바 노출 제어** — 현재 전체 노출 → 역할(조직 배정 단위)에 따라 필요 항목만 표시
3. **임원 역할은 조직 배정 단위 관리** — 동일 인물이 조직별로 최상위/차순위 권한이 다를 수 있음
4. **차순위 임원 = 팀장과 동일 권한** — 팀원 평가·자기 평가만, 평가등급 확정 불가
5. **자기평가 세분화** — 과제업무 / 일반업무 두 가지 의견 입력 구분
6. **평가등급확정 시 팀장/팀원 분리** — 팀장과 팀원을 각각 독립적으로 등급 확정 + 의견 작성
7. **임원 평가등급확정 화면** — 팀원 육성면담서 확인 시 해당 팀원에 대한 팀장 의견도 함께 표시
8. **평가결과 공개는 HR 수동 트리거** — 자동 공개 로직 없음, HR관리자가 공개 시점 직접 설정
9. **목표관리 연동** — 자기 평가 화면에서 기존 목표관리 시스템 데이터 불러오기 API 연동 필요

## 코딩 컨벤션

- 언어: 한국어 UI, 영어 코드/변수명
- 컴포넌트: PascalCase (`GoalCard.tsx`)
- 훅: camelCase + use prefix (`useAuth.ts`)
- Firestore 문서 ID: Firebase Auth UID 또는 자동 생성 ID
- 역할 체크는 반드시 `userProfile.role` 사용
- 동적 라우트(`[id]`)는 `generateStaticParams`에 placeholder `{ id: '_' }` 포함 필요 (static export 조건)
- Sidebar active 판별: `exact: true` 항목은 pathname 정확 일치만 적용 (trailingSlash 처리 포함)
